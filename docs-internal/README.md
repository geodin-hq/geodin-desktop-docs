# docs-internal

Working notes, audits, and product/commercial context that should not appear on docs.geodin.com.

GitBook publishes only `public/en/` (configured in `public/en/.gitbook.yml`), so anything in this folder lives in the GitHub repo for context but is not rendered to readers.

## Current contents

- `AUDIT-import-duplication.md` — 2026-05-06 audit of duplicated auto-assembled content; explains root cause and prevention rules.
- `human-in-the-loop-priorities.md` — prioritised list of docs work that needs Looms, editorial decisions, or original content from a human (e.g. Leapfrog export page, empty stubs in SUMMARY).
- `migration-gaps.md` — known feature gaps (e.g. layer-data import for G1) tracked internally; not for public docs because they read as critical missing features and undermine the marketing narrative.
