# Object Frames

## Object Frames

**Object Frame** is the first required element on any layout; it connects the layout to actual object data via drag-and-drop from the tree. <!-- src: transcript/forms-templates-customization#object-frames -->

Two types of object frame: **Single Object Frame** displays data from one borehole at a time; **Multi-Object Frame** displays data from multiple boreholes simultaneously (e.g., 5 borehole logs side by side). <!-- src: transcript/forms-templates-customization#object-frames -->

Convert between single and multi object frame via right-click > Convert Object Frame. <!-- src: transcript/forms-templates-customization#object-frames -->

Object Frame sphere icon: white = no data connected, red = data connected (single object frame), multiple spheres = multi-object frame. <!-- src: transcript/forms-templates-customization#object-frames -->

For multi-object frame: hold Command key while dragging to add multiple objects to the data source. <!-- src: transcript/forms-templates-customization#object-frames -->

Object frame position can be set exactly via Properties > Position > X, Y, Width, Height (e.g., 0,0 with 21x29.7 cm = A4 portrait). <!-- src: transcript/forms-templates-customization#object-frames -->

Drag-and-dropping objects during layout editing is for editing-mode preview only; it does not permanently link objects to the template. <!-- src: transcript/forms-templates-customization#object-frames -->

When a multi-object frame is used, its data source can be defined to specify exactly which boreholes to include in the output. <!-- src: transcript/forms-templates-customization#object-frames -->
