# Rest API Overview

## REST API / Public API Capabilities

GeoDin has no public REST API discussed in available transcripts; integration is primarily file-based and plugin-based. <!-- src: transcript/api-integrations#rest-api-public-api-capabilities -->

SQL Server client-server databases are supported as a backend — the database connection string must be configured by the customer IT team (documented in the manual). <!-- src: transcript/api-integrations#rest-api-public-api-capabilities -->

GeoDin Onsite has no native server/API and cannot talk directly to a GeoDin server; all integration with the back office is file-based via GeoDin ML and shared folders. <!-- src: transcript/api-integrations#rest-api-public-api-capabilities -->
